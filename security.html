Enterprise-grade security measures to protect your data and API access

Data Protection

üíæ Encrypted Database

All data is stored in a PostgreSQL database hosted on Railway with encryption at rest. Database access is restricted to authorized personnel only.

üîí Password Security

User passwords are hashed using bcrypt before storage. We never store passwords in plain text and cannot retrieve your password if forgotten (password reset required).

üíø Regular Backups

Automated daily backups with 30-day retention. Backups are encrypted and stored separately from production data. We can restore data to any point within the retention period.

üí≥ Stripe PCI Compliance

All payments are processed through Stripe, which is PCI DSS Level 1 compliant (the highest level of payment security). We never see, store, or process your credit card information directly.

Incident Response

In the event of a security incident:

  • We will notify affected users within 72 hours
  • We will provide details about what data was affected
  • We will take immediate steps to contain and remediate the issue
  • We will conduct a post-incident review and implement improvements

Security Contact

If you discover a security vulnerability or have security concerns:

  • Email: security@liendeadline.com
  • Response Time: We aim to respond to security reports within 24 hours

Please do not publicly disclose vulnerabilities until we've had a chance to address them.

Security Best Practices for API Users

  • Protect Your API Key: Never commit API keys to version control or share them publicly
  • Use HTTPS: Always make API requests over HTTPS (never HTTP)
  • Rotate Keys Regularly: Regenerate API keys periodically or if compromised
  • Monitor Usage: Review API usage logs regularly for suspicious activity
  • Implement Rate Limiting: Add client-side rate limiting to prevent accidental abuse
  • Validate Input: Validate all input before sending to our API
