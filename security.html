Security

How we protect your data and ensure system security

Data Encryption

  • • In Transit: All data transmitted between your browser and our servers uses TLS 1.3 encryption (HTTPS)
  • • At Rest: Sensitive data stored in our database is encrypted using industry-standard encryption algorithms
  • • Payment Data: Credit card information is processed securely by Stripe and never stored on our servers
  • • API Keys: API keys are hashed and stored securely. Never share your API key publicly.

Infrastructure Security

  • • Hosting: Our servers are hosted on Railway.app, which provides enterprise-grade security and compliance
  • • Database: PostgreSQL database with encrypted connections and regular backups
  • • Access Control: Multi-factor authentication for admin accounts and restricted API access
  • • Monitoring: 24/7 monitoring for security threats and system anomalies
  • • Updates: Regular security patches and dependency updates

Authentication & Authorization

  • • Password Security: Passwords are hashed using bcrypt before storage
  • • Session Management: Secure session tokens with expiration and rotation
  • • API Authentication: API keys required for all API endpoints
  • • Rate Limiting: Protection against abuse and brute-force attacks
  • • IP Tracking: Calculation limits tracked by IP + user agent hash to prevent abuse

Data Privacy

  • • Data Minimization: We only collect data necessary to provide our service
  • • No Data Selling: We never sell your data to third parties
  • • Data Retention: Data is retained only as long as necessary for service provision
  • • Right to Deletion: You can request deletion of your data at any time
  • • GDPR Compliance: We comply with GDPR and other data protection regulations

See our Privacy Policy for more details.

Payment Security

  • • PCI Compliance: All payments processed through Stripe, which is PCI DSS Level 1 compliant
  • • No Card Storage: We never see or store your full credit card number
  • • Secure Checkout: All payment forms use Stripe's secure hosted checkout
  • • Fraud Detection: Stripe's advanced fraud detection protects against unauthorized transactions

Reporting Security Issues

If you discover a security vulnerability, please report it responsibly:

  • • Email: security@liendeadline.com
  • • Include details about the vulnerability and steps to reproduce
  • • We will respond within 48 hours and work with you to resolve the issue
  • • Please do not publicly disclose vulnerabilities until we've had a chance to fix them

Compliance

  • • GDPR: Compliant with General Data Protection Regulation
  • • CCPA: Compliant with California Consumer Privacy Act
  • • SOC 2: Our hosting provider (Railway) maintains SOC 2 Type II compliance
  • • Regular Audits: We conduct regular security audits and assessments
